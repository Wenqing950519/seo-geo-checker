# 網站健檢與 SEO/GEO 報告模板

報告語言跟隨 `config.language`。資料收集達到最低門檻後即可產出:
至少需要 `ai-positioning` 加上 `seo-audit` 或 `geo-test` 其中一項。
各節沒有數據就寫「本期無數據」或「本期未測」,不要編造。

```markdown
# SEO/GEO 網站健檢報告 — YYYY-MM-DD

## 本期重點(3 點以內)
最值得注意的變化,好壞都算。給「所以呢」而不只是數字。

## 網站定位診斷
- AI/搜尋引擎目前如何理解本站:
- 與品牌自我定位的落差:
- 被錯誤理解或沒有被理解的地方:
- 目前應該納入競品分析的對象:
- 信心等級與資料限制:

## 策略判斷
- 哪些變化真的影響高價值 ICP / 優先 offer
- 哪些只是低價值流量或雜訊
- 本期最需要補的證據或定位缺口

## AI 引用(GEO)
- 整體引用:X/N 次(上期 Y/N)— 樣本量表述規則見 geo-checklist「誠實規範」,
  小樣本用次數不用百分比,±1 次內的變化標示為「測量噪音範圍內」
- 若使用 source_visibility_proxy,分開列示,不可寫成 AI 引用率
- 新增被引用的問題 / 失去引用的問題
- 競品對照:誰在我們沒被引用的題目上被引用了,**以及為什麼贏**
  (引 competitor_win_analysis:贏因維度 + 可複製/不可複製)

## 測試限制(固定揭露,不可省略)
- 樣本量:N 題 × M runs,共 X 次;provider 覆蓋與 proxy 比例
- 本期是否有已知的 provider 模型更新
- 這個樣本量能支持的結論層級(方向參考/初步證據/可信趨勢)

## 技術 SEO
- 問題數:critical X / warning Y(上期對照)
- 本期修掉的、新出現的
- 對收錄、排名、AI 可引用性的影響

## 內容
- 待審草稿:N 份(列檔名)
- 已發布內容的初步表現(若可知)
- 缺少的高價值頁面、比較頁、FAQ、證據頁或案例頁

## SEO 專家優化方向
每一項都要包含:目標頁/新資產、要解決的查詢或 AI 問題、依據、具體改法、預期影響、不確定性。

| 優先級 | 類型 | 目標頁/資產 | 問題或機會 | 建議動作 | 依據 | 預期影響 |
```

## 因果用語規範

解讀任何變化前,先對照 `data/changes.log`:

| 情況 | 允許的措辭 |
|------|-----------|
| 變化無對應變更記錄 | 「觀察到變化,原因不明,可能為隨機波動或模型更新」 |
| 變化與某變更同期(單期) | 「與 X 變更同期出現,可能相關,待下期驗證」 |
| 連續兩期以上同方向 + 有變更記錄 | 「初步證據支持與 X 相關」 |
| 任何情況 | 禁用「因為/導致/證明了」 |

## Telegram 摘要格式

推播只送精簡版:本期重點 + 定位/引用率變化 + P1 待辦提醒,150 字內,附完整報告檔名。
