# 方法論與信任區

## 區塊目標

目前如果還沒有公司案例、完整團隊與成熟顧問資歷，不要硬包裝。這一區用「透明方法論」建立信任。

## 區塊標題

```text
用透明的方法，檢查 AI 是否真的理解你的網站
```

## 區塊副標

```text
這套流程不是為了製造漂亮分數，而是把網站主常看不見的 SEO/GEO 問題拆開：搜尋引擎能不能讀取、AI 能不能理解、內容能不能被引用，以及下一步應該先修哪裡。
```

## 關於這套流程

```text
這套健檢流程由獨立 SEO/GEO 研究者設計，目標是把網站健檢流程產品化。自動化工具負責收集公開資料與整理初步訊號，最終優化方向仍以人工判讀、搜尋意圖與實際商業情境為準。
```

## 方法論原則

### 1. 先看定位，再看競品

```text
競品不只來自業主自認的名單，也來自 AI 與搜尋結果實際把誰放在同一個答案場景中。因此流程會先判斷 AI 眼中的網站定位，再延伸到競品與內容缺口。
```

### 2. SEO 基礎仍然重要

```text
GEO 不是取代 SEO。AI 搜尋通常仍依賴可被搜尋引擎讀取、理解與信任的內容。技術 SEO、內容品質、結構化資料與品牌實體訊號仍是基礎。
```

### 3. 不承諾保證引用

```text
AI 搜尋結果會受到模型、時間、地區、搜尋環境與資料來源影響。這份健檢會盡量標示測試限制與信心等級，不會承諾保證被 AI 引用或保證排名。
```

### 4. 建議必須具體

```text
好的 SEO/GEO 建議不應該只是「增加 FAQ」或「提升內容品質」。報告會盡量指出目標頁面、要回答的問題、背後原因與優先級。
```

## 可以放在頁面上的短版

```text
這份健檢結合技術 SEO 檢查、AI 眼中定位分析與內容可引用性評估。它不是完整顧問 audit，而是一份幫你快速看見問題的初步診斷：AI 是否理解你、搜尋引擎是否讀得懂你、內容是否有機會被引用，以及下一步應該先修哪裡。
```

## Algorithm V3 實測方法

GeoCheck 的 GEO 主分數以 Perplexity 固定查詢集實測為核心：每站一次精確實體驗證、兩次非品牌探索，再搭配內容可引用性與必要技術存取。Perplexity 無法量測時顯示未知，不以站內結構補分。Gemini 只負責解讀或研究分類，不參與分數。

白皮書模式每站使用 3 次 Perplexity 與 1 次 Gemini Flash-Lite；Gemini 只分類基本資訊、產業與結構，不產生優化建議。所有研究結果須附模型、日期、查詢集、失敗率與資料集雜湊。
