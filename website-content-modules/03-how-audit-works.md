# 這份健檢如何運作

## 區塊目標

這一區用來建立信任。它要回答用戶心中的疑慮：

- 這份報告是不是 AI 亂寫的？
- 到底檢查了什麼？
- 為什麼我可以相信初步結果？
- 免費健檢和完整顧問服務差在哪？

```text

```

```

```

## 區塊副標

```text
輸入網址後，系統會先讀取網站的公開資訊，判斷搜尋引擎與 AI 可能如何理解你的品牌、服務與內容結構。接著再檢查技術 SEO、內容可引用性與 AI 搜尋能見度，最後整理出最值得優先修正的方向。
```

## 五步驟流程

### 1. 讀取網站公開資訊

說明：

```text
我們會檢查首頁、主要 metadata、heading、schema、robots.txt、sitemap 與 llms.txt，了解搜尋引擎能否順利讀取你的網站。
```

使用者會得到：

```text
一份網站基礎可讀性與技術訊號的初步檢查結果。
```

### 2. 分析 AI 眼中定位

說明：

```text
系統會根據網站內容判斷：你屬於什麼品類、服務哪些受眾、解決什麼問題，以及是否容易被 AI 誤解。
```

使用者會得到：

```text
AI / search 可能如何理解你的網站，以及這個理解是否和你想傳達的定位一致。
```

### 3. 檢查 SEO/GEO 基礎訊號

說明：

```text
我們會檢查 title、meta description、H1、結構化資料、內容段落、FAQ、可引用證據等，找出影響搜尋與 AI 理解的問題。
```

使用者會得到：

```text
影響搜尋收錄、內容理解與 AI 可引用性的具體問題清單。
```

### 4. 生成 AI 搜尋測試問題

說明：

```text
系統會根據你的網站定位，產生幾個使用者可能會問 AI 的問題，例如「如何選擇...」、「有哪些推薦...」、「哪個服務適合...」。
```

使用者會得到：

```text
一組更接近真實搜尋情境的 GEO 測試問題，而不是套用所有網站都一樣的固定題目。
```

### 5. 彙整優先修正方向

說明：

```text
報告會把問題整理成 P1-P3 優先級，讓你知道應該先修技術結構、內容可引用性，還是品牌定位訊號。
```

使用者會得到：

```text
三個最值得優先處理的方向，包含問題依據、建議方向與預期影響。
```

## 透明說明 / 限制聲明

```text
這是一份初步健檢，不等於完整 SEO audit。AI 搜尋結果可能受到時間、地區、模型版本與搜尋環境影響，因此報告會標示資料限制與信心等級。若需要完整優化，仍建議結合 Search Console、網站後台數據、競品研究與人工顧問判讀。
```
