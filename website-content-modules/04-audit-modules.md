# 五大健檢模組

## 區塊目標

讓用戶清楚知道工具會檢查什麼，並理解這不是一般 SEO checklist，而是從 AI 定位、技術、內容與商業優先級一起看。

## 區塊標題

```text
五大模組，完整診斷你的 AI 搜尋能見度
```

## 區塊副標

```text
不只是技術掃描。這份健檢會先確認 AI 如何理解你的網站，再檢查 SEO 基礎、內容可引用性與最值得優先修正的機會。
```

## 模組 1：AI Positioning

標題：

```text
AI 眼中定位
```

英文標籤：

```text
AI POSITIONING
```

說明：

```text
先確認 AI 與搜尋引擎目前怎麼理解你的網站：你屬於什麼品類、服務哪些受眾、解決什麼問題，以及和哪些競品同場出現。
```

檢查項目：

- 品類辨識
- 受眾理解
- 核心服務
- 使用情境
- 競品共現
- 定位一致性

## 模組 2：Technical SEO

標題：

```text
技術 SEO 健檢
```

英文標籤：

```text
TECHNICAL SEO
```

說明：

```text
檢查搜尋引擎與 AI 是否能順利讀取網站基礎訊號，包含 title、meta、heading、canonical、schema、robots.txt、sitemap 與 llms.txt。
```

檢查項目：

- title
- meta description
- H1 / heading hierarchy
- canonical
- robots.txt
- sitemap.xml
- structured data
- Open Graph
- llms.txt
- indexability

## 模組 3：Content Citeability

標題：

```text
內容可引用性
```

英文標籤：

```text
CONTENT CITEABILITY
```

說明：

```text
判斷網站內容是否容易被 AI 摘錄、引用與轉述。AI 需要清楚定義、具體證據與可獨立回答問題的內容，而不只是漂亮的行銷文案。
```

檢查項目：

- 定義型段落
- FAQ
- 具體數據
- 案例與佐證
- 作者與更新日期
- 來源與引用
- 比較表
- 決策型內容

## 模組 4：GEO Visibility

標題：

```text
GEO 能見度測試
```

英文標籤：

```text
GEO VISIBILITY
```

說明：

```text
依據網站定位與產業情境，生成 5-8 題使用者可能會問 AI 的問題，觀察品牌是否出現、是否被引用，以及哪些競品在同樣問題中被提到。
```

檢查項目：

- 產業問題生成
- 品牌是否出現
- 是否被引用
- 競品出現紀錄
- AI / search source visibility
- 信心等級

## 模組 5：Opportunity Summary

標題：

```text
優先修正方向
```

英文標籤：

```text
OPPORTUNITY SUMMARY
```

說明：

```text
報告最後會把問題整理成三個最高優先級的修正方向，讓你知道應該先處理技術結構、內容可引用性，還是品牌定位訊號。
```

輸出項目：

- P1-P3 優先級
- 目標頁面或新資產
- 問題依據
- 建議方向
- 預期影響
- 不確定性
