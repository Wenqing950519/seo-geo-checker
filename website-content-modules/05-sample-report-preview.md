# 報告範例區

## 區塊目標

讓用戶在提交網址前先知道會拿到什麼。這一區要比一般 landing page 更具體，因為它是建立信任與轉換的核心。

## 區塊標題

```text
你會拿到一份這樣的網站健檢報告
```

## 區塊副標

```text
報告不只給分數，而是說明 AI 如何理解你的網站、哪些 SEO/GEO 問題正在影響能見度，以及最值得優先修正的方向。
```

## 範例情境

```text
範例網站：B2B SaaS 服務網站
```

```text
網站目標：讓中小企業主找到一套能簡化內部流程的 SaaS 工具。
```

## Overall Health Score

分數：

```text
62 / 100
```

狀態：

```text
Needs Work
```

說明：

```text
這個網站的技術 SEO 基礎大致可被搜尋引擎讀取，但 AI 對品牌定位的理解仍偏模糊。網站內容有介紹產品功能，卻缺少能回答決策型問題的段落、比較內容與可引用證據，因此在高意圖問題中容易輸給定位更清楚的競品。
```

## AI 眼中定位摘要

```text
AI / search 可能將此網站理解為一般專案管理或營運工具，但無法明確判斷它最適合哪種產業、團隊規模或使用情境。
```

定位缺口：

- 服務對象不夠明確
- 首頁缺少一句能被 AI 摘錄的品牌定義
- 缺少和競品比較的決策型內容
- 缺少第三方佐證與案例

## 技術 SEO 摘要

發現問題：

- 部分頁面 meta description 過於行銷化，缺少搜尋意圖對應
- 首頁缺少 Organization schema
- 服務頁缺少 FAQPage 或 Service schema
- sitemap 可讀，但重要服務頁層級不夠清楚
- 未提供 llms.txt

## 內容可引用性摘要

```text
網站有描述功能，但缺少能被 AI 直接引用的「答案型內容」。例如：什麼情境適合使用、和其他工具差在哪、導入需要多久、常見成本與限制等。
```

主要缺口：

- 缺少 FAQ
- 缺少比較表
- 缺少數據或案例佐證
- 缺少決策階段內容

## GEO 測試摘要

測試問題範例：

- 小型企業如何選擇流程管理工具？
- 有哪些適合中小企業的專案管理 SaaS？
- 導入營運管理工具前要注意什麼？

結果摘要：

```text
在測試問題中，網站品牌未被穩定提及。競品較常出現在「推薦工具」與「比較選項」類問題中，原因可能是競品擁有更清楚的比較頁、案例頁與第三方引用。
```

## P1-P3 優先修正方向

| 優先級 | 類型 | 目標 | 建議方向 |
|--------|------|------|----------|
| P1 | AI Positioning | 首頁 / About | 補上一段清楚定義品牌、服務對象與核心使用情境的文字，讓 AI 能判斷網站屬於哪個品類 |
| P2 | Technical SEO | 全站 | 新增 Organization、WebSite、Service schema，提高搜尋引擎與 AI 對品牌實體的辨識 |
| P3 | Content | 服務頁 / 新文章 | 建立決策型 FAQ 與比較表，回答「適合誰、和競品差在哪、導入成本」等高意圖問題 |
